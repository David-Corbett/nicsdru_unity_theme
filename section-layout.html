2 layout Layout

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

The theme provides a number of Drupal 'regions' namely:-

  • header: 'Site branding'
  • search: 'Search'
  • primary_menu: 'Main menu'
  • slideshow: 'Slideshow'
  • front_welcome: 'Front welcome'
  • top_first: 'First top'
  • top_second: 'Second top'
  • top_third: 'Third top'
  • breadcrumb: 'Breadcrumb'
  • sidebar_first: 'First sidebar'
  • sidebar_second: 'Sidebar second'
  • content: 'Content'
  • bottom_widget_1: 'Bottom widget 1'
  • bottom_widget_2: 'Bottom widget 2'
  • bottom_widget_3: 'Bottom widget 3'
  • footer_first: 'First Bottom'
  • footer_second: 'Second Bottom'
  • footer_third: 'Third Bottom'
  • footer_fourth: 'Fourth Bottom'
  • footer: 'Footer Bottom'

Code in the src/scss/layout directory of the theme primarily facilitates structuring and placement of these Drupal regions on a given page. Demonstration templates are provided below. Some regions are configured to only render on the front page of the site (these are outlined further in the documentation in the regions section of this page. Note that a Drupal region will only render if there is some code / markup in the region for Drupal to output. See here for more information on Drupal regions.

2.1 layout.header Header

The header layout partial deals with the structure of elements in the header area of the page, this typically includes the:-

  • Site Branding region which renders (if available)
    • site_logo: Logo for site as defined in Appearance or theme settings.
    • site_slogan: Slogan for site as defined in Site information settings.
  • Search region which is a region intended to contain a site search block (if site search is configured).
Source: scss/2_layout/_header.scss, line 6

2.2 layout.container Container

The container layout partial deals with the structure of elements in the main area of the page, this typically includes the:-

  • The main content region.
  • The breadcrumb region.
  • Sidebar first region.
  • Sidebar second region.

The layout of the sidebar sections is controlled by the inclusion of a class or classes via a preprocess hook on the <body> tag of the page. The width of the sidebar regions / areas can be overridden by declaring $variables - XXX

2.3 layout.regions Regions

The regions layout partial deals with the structure of elements in regions of the page not otherwise defined, so this includes the:-

  • The front welcome region.
  • The first top region.
  • The second top region.
  • The third top region.

2.4 layout.footer Footer

The footer layout partial deals with the structure of elements in the footer area of the page, this typically includes the:-

  • The footer first region.
  • The footer second region.
  • The footer third region.
  • The footer fourth region.
  • The footer bottom region.

Note that background colour for the footer area is controlled by variables in the init/colors partial. You can view the currently defined color values here in the styleguide.
