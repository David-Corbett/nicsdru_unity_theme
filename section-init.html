4 init Initialization partial

To make it easier to use all variables and mixins in any sass file in this project, each .scss file has a @import 'init'; declaration. The init/_index.scss file is in charge of importing all the other partials needed for the project.

The initialization partial is organized in this way:

  • First we set any shared sass variables.
  • Next we import third party sass modules / libraries.
  • Last we define our custom functions and mixins for this project.

4.1 init.variables Variables

Variables are values assigned to a name that begins with $, which can then be referred to that name instead of the value itself. Despite their simplicity, they're one of the most useful tools Sass brings to the table. Variables make it possible to reduce repetition, do complex math, configure libraries, and much more.

4.1.1 init.variables.colors Colors

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

The color() function can be used to add colors to CSS properties. To learn more, read the Chroma documentation. When using a IDE for writing code, declaring a function will not make the options for the function available, therefore using the color() function becomes more difficult as the developer must remember the available values and type them in full etc, therefore for each color value, there is an equivalent variable with the same name - the variable should be set using chroma's color() function to retain the capabilities of the sass module. Declarations in scss code for color variables should be declared using the $variable method.

4.1.2 init.variables.breakpoints Breakpoint variables

Set some values for where responsive changes occur - the values are set for use with min-width media queries.

Parameters:
  • $mob
    a mobile device view in landscape.
  • $phab
    a tablet sized device being viewed portrait;
  • $tab
    a tablet sized device being viewed landscape;
  • $lap
    a larger device with a smaller screen such as a laptop;
  • $desk
    a full sized screen, able to accommodate the largest sizes currently available;
  • $max-grid-width
    sets the maximum value the content area can expand to, on larger devices, the content container is centered on the page.

4.1.3 init.variables.vertical-rhythm Vertical rhythm variables

Set some values that

Parameters:
  • $base-font-size
    The font size set on the root html element.
  • $base-line-height
    The base line height determines the basic unit of vertical rhythm.
  • $rhythm-unit
    The length unit in which to output vertical rhythm values. Supported values: px, em, rem.
  • $rem-with-px-fallback
    Whether to output fallback values in px when using rem as the rhythm-unit.
  • $round-to-nearest-half-line
    Round the line height to the nearest half line height instead of the nearest integral line height.
  • $indent-amount
    The amount lists, blockquotes and comments are indented.

4.1.4 init.variables.typography The default font family Font faces, stacks and sizes.

Font styling and line heights are controlled by the several variables that are used by mixins like adjust-font-size-to(). These variable and mixins are documented in Compass. We also use a custom mixin responsive-typo-map() to make responsive typography much more manageable using sass maps. We import Google fonts in the head of the document and set the base family to be 'Open Sans', sans-serif

Parameters:
  • $normal-text
    defines a responsive typography map for standard paragraph text in the document.
  • $h1-text
    defines a responsive typography map for normal heading 1 text in the document.
  • $h2-text
    defines a responsive typography map for normal heading 2 text in the document.
  • $h3-text
    defines a responsive typography map for normal heading 3 text in the document.
  • $h4-text
    defines a responsive typography map for normal heading 4 text in the document.
  • $h5-text
    defines a responsive typography map for normal heading 5 text in the document.
  • $h6-text
    defines a responsive typography map for normal heading 6 text in the document.
